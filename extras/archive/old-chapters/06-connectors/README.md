## この章で学ぶこと

- MCP（Model Context Protocol）が「AI 版の API」だという捉え方（実例: Notion / Slack / Gmail 連携）
- Skill が「特定の作業手順をまとめた能力」だという捉え方（実例: add-task / ai-news-briefing / screenshot-to-calendar / tag-memos）
- コマンド・サブエージェント・skill・MCP の使い分け（いつどれを選ぶか）
- 自分の作ったものを「なぜこの形にしたか」で振り返る
