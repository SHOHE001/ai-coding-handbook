# Chapter 1: Claude Code とは何の道具か

> Status: 完了

## この章で学ぶこと

- Claude Code の正体は「賢い検索エンジン」でも「魔法の自動化マシン」でもないこと
- その中身である **LLM** が、実は「次の単語を確率で当てているだけ」だということ
- だから何が得意で、何が苦手なのか
- なぜ「曖昧な指示」を投げると事故るのか
- このハンドブック全体で何を順番に言語化していくのか

## 前提知識

特になし。Claude Code を一度でも触ったことがあれば十分です。

## 本文

### 1. まず、よくある 2 つの誤解をほどく

Claude Code を使い始めると、無意識にこの 2 つのどちらかのイメージを持ちがちです。

- **誤解A「賢い検索エンジン」**: 質問すれば正しい答えがどこかから出てくる箱
- **誤解B「魔法の自動化マシン」**: 指示すれば何でも完璧にこなしてくれる万能ロボット

どちらも実態とずれています。正しくは、<mark>**Claude Code は「言葉で指示できる、ものすごく物知りだが思い込みも激しい新人アシスタント」**</mark>に近い。物知りだけど、知ったかぶりもするし、こちらの言葉足らずを勝手に補完して的外れなことをする。この「人間っぽいクセ」を理解することが、使いこなしの第一歩です。

このクセがどこから来るのか。それは中身の **LLM** の仕組みに理由があります。

### 2. LLM の正体は「次の単語の予測機」

Claude Code の頭脳は **LLM（Large Language Model、大規模言語モデル）** というものです。やっていることを一言で言うと、

<mark>**「ここまでの文章の続きとして、次に来そうな単語を確率で選ぶ」**</mark>

これをひたすら繰り返しているだけ。たとえば「今日は天気が」まで入力されたら、続きとして「いい（高確率）」「悪い（高確率）」「カレー（低確率）」…と候補に確率を振って、それっぽいものを選ぶ。選んだら、それを含めてまた次の単語を予測する。この繰り返しで長い文章が出てきます。

**身近な実例**: スマホで文字を打つときの **予測変換** を思い出してください。「おは」と打つと「おはよう」が候補に出る、あれの超強力版だと思うとイメージしやすい。違いは、スマホは直前の数文字しか見ていないのに対し、LLM は **会話まるごと・大量の文書から学んだパターン** を踏まえて次を予測する点です。

> [!CAUTION]
> ここが全ての出発点。<mark>**LLM は「正しいことを答えよう」としているのではなく、「それっぽく続く言葉を選んで」いる**</mark>。たいていは正しいことがそれっぽいので正解になるが、**間違ったこともそれっぽく言える**。この性質を忘れると、後の章のほぼ全部が「なぜ？」になる。

### 3. だから「知らないことも自信満々に言う」

LLM が間違うとき、人間のように「うーん、自信ないなあ」と濁ることは基本ありません。<mark>**間違った内容も、正しい内容とまったく同じ堂々とした口調で出てくる**</mark>。これは「それっぽい続き」を選んでいるだけだから当然で、文章の自信ありげなトーンと、中身の正しさは別物なのです。

この「もっともらしい嘘」を **ハルシネーション（hallucination、幻覚）** と呼びます。

**身近な実例**: 存在しない関数名や、書いてもいないファイルの中身を、さも当たり前のように言ってくることがあります。Claude Code が「`config.json` の設定を変えますね」と言うが、そんなファイルは存在しない ── みたいなやつ。これは嘘をついているのではなく、「この文脈なら `config.json` がありそう」という確率的な思い込みで喋っているだけ。

だから実務では <mark>**「出力をうのみにせず、自分の目で確認する」**</mark>が大前提になります。とくにファイルの中身・コマンドの挙動・事実関係は、必ず実物を見て裏を取る。

### 4. 何が得意で、何が苦手か

確率的な予測機だと分かると、得意・不得意がきれいに説明できます。

| 得意なこと | なぜ |
|---|---|
| 定型的な文章・コードの生成 | 世の中に大量のパターンがあり、予測しやすい |
| 言い換え・要約・翻訳 | 「同じ意味の別の言い方」はパターンの宝庫 |
| 退屈な繰り返し作業 | 飽きないし、文句も言わない |
| たたき台を高速で出す | 0 → 1 が速い。質より速さが要る場面に強い |

| 苦手なこと | なぜ |
|---|---|
| 正確な計算・厳密な数え上げ | 「それっぽい数字」を出してしまう。電卓ではない |
| 最新の事実 | 学習した時点までの知識。今日の出来事は知らない |
| 「知らない」と言うこと | 確率的に何か出してしまうので、空欄が苦手 |
| こちらの意図の読心 | 書いていないことは（それっぽく）推測で埋める |

**身近な実例**: しょうへいさんが MAGI（複数のサブエージェントに多数決させるコードレビュー）を作ったのは、まさにこの「1 回の出力をうのみにしない」発想の実装です。1 人の新人の意見より、3 人に別々に見せて多数決を取ったほうが事故りにくい ── LLM のクセを仕組みで補っている良い例。

### 5. なぜ「曖昧な指示」は事故るのか

LLM は、こちらが **書かなかったこと** を勝手に推測で埋めます。悪気はなく、「それっぽい続き」を選ぶ性質上そうなる。だから指示が曖昧だと、埋めた推測がこちらの意図とずれて事故ります。

```
曖昧な指示:  「いい感じにして」
→ 「いい感じ」の中身を LLM が勝手に決める。たいてい期待と違う

具体的な指示: 「この関数のエラーメッセージを日本語にして。
              処理のロジックは変えないで」
→ 推測の余地が小さい。事故りにくい
```

> [!CAUTION]
> <mark>**指示の曖昧さ = LLM に与える「勝手に推測していい余白」の広さ**</mark>。余白が広いほど、出力は自分の意図から遠ざかる。これが Chapter 2（プロンプト）と Chapter 3（CLAUDE.md）の全部の動機。「ちゃんと伝える」のは礼儀ではなく、事故を減らす技術。

**身近な実例**: しょうへいさんが「実装を委ねる前に、要件を構造化した Q&A で絞り込む」習慣を持っているのは、この余白を狭める作業そのもの。経験的に正解にたどり着いている。このハンドブックは、その「なぜ効くのか」を後から言葉にしていきます。

### 6. Claude Code は「LLM ＋ 道具」

ここまで LLM 単体の話をしてきましたが、Claude Code は LLM に **手足** を付けたものです。LLM はそのままだと「喋るだけ」ですが、Claude Code は LLM が

- ファイルを読む・書く
- コマンドを実行する
- 外部サービス（Notion など）を操作する

といった **実際の行動** を取れるようにしています。「考える頭（LLM）」に「ファイルやコマンドを触る手（ツール）」を付けて、頭が手に指示を出しながらタスクを進める ── これが Claude Code の基本構造です。

**身近な実例**: しょうへいさんの /zenn-log（学習ログを Zenn に投稿するコマンド）も、add-task（Notion にタスクを登録する skill）も、「LLM が考えて、手を動かす」の具体例。頭だけでも手だけでも成り立たず、両方そろって初めてツールになります。

この「頭」と「手」の関係は、Chapter 5（拡張）と Chapter 6（外部接続）で深掘りします。

これからどこへ向かうのかを地図にしておきます。全章 **読み物中心**、手を動かす実装パートは置きません。

| Ch | テーマ | 一言で言うと |
|---|---|---|
| 1 | Claude Code とは何の道具か（今ここ） | 相手の正体とクセを知る |
| 2 | 指示の出し方 | 推測の余白を狭める技術 |
| 3 | CLAUDE.md 設計 | プロジェクトの記憶を書く |

## 実務のコツ

- <mark>**出力は必ず一度疑う**</mark>: とくにファイルの中身・コマンドの結果・事実は、自分の目で確認してから信じる。「自信ありげな口調」は正しさの保証にならない
- **得意なことを任せる**: 0 → 1 のたたき台、退屈な繰り返し、言い換え。ここは強い。逆に正確な計算や最新の事実は別の手段で裏を取る
- **曖昧さは敵**: 「いい感じに」より「○○を△△して、××は変えないで」。余白を残さない
- **困ったら新人アシスタントだと思う**: 物知りだが思い込みが激しい新人に頼むとき、自分なら何をどこまで説明するか ── それがそのまま良い指示になる

## つまずきポイント

> [!CAUTION]
> 入門時点で **誤解しがちな点のリスト**。とくに 1 番目は、このハンドブック全体の土台になる。

- <mark>**LLM は「正しさ」ではなく「それっぽさ」で動く**</mark>: 正しい答えを検索しているのではなく、続きを予測している。だいたい正しいが、間違いも堂々と出る
- **「自信ありげ ＝ 正しい」ではない**: トーンと中身の正しさは無関係。淡々と嘘を言うこともある
- **「知らない」を言わせるのは難しい**: 空欄が苦手なので、知らないことも何か埋めてしまう。「分からなければ分からないと言って」と明示的に頼むと多少マシになる
- **Claude Code の細かい機能はバージョンで変わる**: コマンド名や挙動は更新されることがある。このハンドブックでも、細部は「公式ドキュメントで確認」と添える方針。仕組みの理解は古びないが、操作の細部は古びる
