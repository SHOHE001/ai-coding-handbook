## 本文

### 1. なぜ「演出」が効くのか

しょうへいさんは「演出が使用継続の動力になる」タイプ。MAGI をエヴァの3賢者になぞらえたのも、その表れです。<mark>**道具は、見た目や手触りが好きだと、使い続けられる**</mark>。Claude Code にも、見た目や応答スタイルを自分好みにする層が用意されています。

| 層 | 何を変える | 設定場所 |
|---|---|---|
| **出力スタイル** | Claude の応答の仕方・persona | `~/.claude/output-styles/` |
| **ステータスライン** | 画面下の情報表示 | `settings.json` の `statusLine` |
| **設定・テーマ** | 配色・言語・モデルなど | `settings.json` |

### 2. 出力スタイル（output styles）

**出力スタイル**は、Claude の応答の仕方を変える仕組みです。Markdown ファイルに「こう振る舞って」と書いて、`~/.claude/output-styles/` に置きます。

```markdown
---
name: Diagrams First
description: 説明はまず図から始める
---

コードや構成を説明するときは、最初に図（Mermaid 等）を示してから、文章で説明してください。
```

公式にも、即実行型の「Proactive」、教えてくれる「Explanatory」、一緒に学ぶ「Learning」などが用意されています。`/config` の Output style から選べます。

> [!CAUTION]
> 出力スタイルは<mark>**セッション開始時に読み込まれます**</mark>。変えてもすぐ反映されないときは、`/clear` か新しいセッションで効きます。フロントマターの項目は版で変わるので、公式の *Output styles* ページで確認してください。

### 3. ステータスライン（statusLine）

**ステータスライン**は、画面下に「今どんな状態か」を出す帯です。`settings.json` に**コマンド型**で指定し、スクリプトが受け取った情報（モデル名・コンテキスト使用率・コストなど）を整形して表示します。

```json
{
  "statusLine": {
    "type": "command",
    "command": "powershell -NoProfile -File C:\\Users\\zereh\\.claude\\statusline.ps1"
  }
}
```

スクリプト（PowerShell）は、標準入力で来る JSON を受け取って文字列を返すだけ:

```powershell
# C:\Users\zereh\.claude\statusline.ps1
$data = $input | ConvertFrom-Json
"[$($data.model.display_name)]  context $($data.context_window.used_percentage)%"
```

<mark>**「今どのモデルで、机がどれくらい埋まっているか」が常に見える**</mark>と、第14章のコンテキスト・コスト管理がぐっとやりやすくなります。演出が実用にもつながる例です。

> [!CAUTION]
> ステータスラインに渡る JSON の正確な項目（`context_window.used_percentage` など）は公式に完全な一覧がない場合があり、版でも変わります。<mark>**まず簡単な表示から試し**</mark>、必要な項目は公式の *Status line* ページで確認してください。Windows では PowerShell の実行ポリシーで止まることがあるので、その場合は実行ポリシーを調整します。

### 4. テーマ・その他の設定

`settings.json` には、見た目や挙動を変えるキーが他にもあります（実在・名称は版で確認）。

- `theme`: 配色（dark / light）
- `language`: 表示言語（しょうへいさんは japanese 運用）
- そのほか、モデルや思考の強さ、自動圧縮の有無など

これらは第3章の settings.json と同じファイルに書きます。<mark>**自分が毎日見る画面を、自分が気持ちよく使える状態に整える**</mark> ── それが継続の燃料になります。

### 5. やりすぎない

> [!CAUTION]
> 演出は楽しいですが、<mark>**凝りすぎて実用を損なわない**</mark>こと。重いステータスラインのスクリプトは表示を遅くし、奇抜すぎる出力スタイルは作業の邪魔になります。「気持ちよく、でも邪魔にならない」が目安。CLAUDE.md の「読みやすさ優先・やりすぎない」と同じ精神です。

## つまずきポイント

> [!CAUTION]
> 演出・カスタマイズでハマりがちな点。

- **出力スタイルが反映されない**: セッション開始時に読まれる。`/clear` か再起動で効く
- **ステータスラインに JSON が生で出る**: スクリプトでパースしていない。`ConvertFrom-Json` や `jq` で処理
- **Windows の実行ポリシー**: PowerShell スクリプトが動かないときは実行ポリシーを調整
- **エンコーディング**: 設定・スクリプトは UTF-8（BOM なし）
- **凝りすぎ**: 重い・奇抜すぎる演出は実用を損なう。ほどほどに
