# 第14章: コンテキスト・コスト戦略

> Status: 完了
> 発展（応用）コース。大きめの作業を、賢く・安く回す

## この章で学ぶこと

- コンテキスト（机の広さ）とコストは表裏一体であること
- 机を軽く保つ運用（`/clear`・`/compact`・必要部分だけ読む）
- サブエージェントで重い作業を**別の机に分離**する
- モデルを**使い分ける**（haiku / sonnet / opus）
- キャッシュを効かせる・コストを監視する

## 本文

### 1. コンテキストとコストは表裏一体

第1章で「会話が長いと、前を忘れ・精度が落ち・コストが増える」と学びました。この3つは**同じ原因**から来ています。<mark>**机（コンテキスト）に情報が積もるほど、毎回それを読み直すので、遅く・不正確に・高くなる**</mark>。

だから「コンテキストを管理すること」と「コストを抑えること」は、ほぼ同じ作業です。本章は、その具体策をまとめます。

### 2. 机を軽く保つ

いちばん基本で、いちばん効くのがこれです。

| やること | 道具 | 効果 |
|---|---|---|
| 話題が変わったらリセット | `/clear` | 古い文脈を捨てて軽くする |
| 長くなったら要約 | `/compact` | 要点を残して圧縮 |
| 巨大ファイルは部分だけ | 必要箇所を指定 | 机を一気に埋めない |

<mark>**長い1本の会話で全部やろうとしない**</mark>。タスクが切り替わったら `/clear`、同じ作業が長引いたら `/compact`。スマホのタブを閉じるのと同じ感覚です。

### 3. 重い作業は「別の机」へ分離する

第5章・第10章で見たサブエージェントは、コスト戦略でも主役です。大量のファイルを読む調査を**メインの机**でやると、検索結果が机を埋めて以降ずっと高くつきます。

<mark>**重い調査は読み取り専用のサブエージェントに丸投げし、要点だけ持ち帰る**</mark>。メインの机はきれいなまま。これだけで、長い作業のコストと精度が大きく変わります。

### 4. モデルを使い分ける

すべてを最上位モデルでやる必要はありません。作業の重さに合わせて選びます。

| モデル | 向いている作業 |
|---|---|
| **haiku**（軽い） | 調査・検索・単純な整形 |
| **sonnet**（標準） | 普通の実装・編集 |
| **opus**（重い） | 難しい設計判断・込み入ったデバッグ |

サブエージェント（第5章）は `model` で指定できるので、<mark>**調査役は haiku、難所だけ opus**</mark>のように振り分けられます。「全部 opus」は、速くも安くもありません。

> [!CAUTION]
> モデル名・指定方法は版で変わります。<mark>**「軽い作業は軽いモデル、難所だけ重いモデル」**</mark>という**考え方**を持っておけば、名前が変わっても応用できます。指定の細部は公式の *Model configuration* で確認してください。

### 5. キャッシュを効かせる

同じ前提（CLAUDE.md やシステム的な指示）を毎回読ませると、本来コストがかさみます。ここで効くのが**プロンプトキャッシュ**です。<mark>**変わらない部分を使い回すと、2回目以降が安くなる**</mark>仕組みで、多くの場合は自動で効きます。

実務的に効かせるコツは、第2章とつながります。<mark>**CLAUDE.md を安定させ（毎回コロコロ変えない）、厳選しておく（150行ルール）**</mark>。変わらない土台が大きいほど、キャッシュの恩恵を受けやすくなります。

### 6. コストを監視する

「気づいたら使いすぎていた」を防ぐには、見えるようにするのが一番です。

- ヘッドレス実行（第9章）の `--output-format json` には `total_cost_usd` が含まれる。これをログに残せば使用料を追える
- 自動・定期実行（第9章）は回数が見えにくいので、頻度と上限を先に決める

> [!CAUTION]
> マルチエージェント（第10章）や定期実行（第9章）は、<mark>**便利さと引き換えにコストが膨らみやすい**</mark>。並列で何体も走らせる前に「本当に必要か」を一拍考える。しょうへいさんの「コストゼロ／最小コスト」原則は、ここで効いてきます。

## つまずきポイント

> [!CAUTION]
> コンテキスト・コストでハマりがちな点。

- **長い会話を放置**: 切らずに続けると遅く・高く・不正確に。`/clear`・`/compact` をこまめに
- **全部を最上位モデル**: 軽い作業まで opus は無駄。使い分ける
- **巨大ファイルの丸ごと貼り付け**: 机が一気に埋まる。必要部分だけ
- **CLAUDE.md を頻繁に書き換え**: キャッシュが効きにくくなる。土台は安定させ厳選
- **並列の乱用**: マルチエージェントは数だけコスト増。必要なときに絞る

## 重要な単語まとめ

| 用語 | 一言で言うと | 身近な実例 |
|---|---|---|
| **コンテキスト＝コスト** | 机が重いほど遅く高く不正確に | 長い会話の放置 |
| **`/clear`・`/compact`** | 机を捨てる・圧縮する | 話題転換でリセット |
| **机の分離** | 重い作業をサブエージェントへ | 調査を別の机で |
| **モデル使い分け** | 作業の重さでモデルを選ぶ | 調査は haiku、難所は opus |
| **プロンプトキャッシュ** | 変わらない前提を使い回して安くする | 安定した CLAUDE.md |
| **コスト監視** | 使用料を見えるようにする | `total_cost_usd` をログ |
