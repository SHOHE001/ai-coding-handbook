# 第0章: 全体地図 — Claude Code の拡張点6つ

> Status: 完了（見本として先行作成）

## この章で学ぶこと

- Claude Code は「素のまま」でも強いが、**6つの拡張点**で自分仕様に育てられること
- その6つ（CLAUDE.md / スラッシュコマンド・スキル / サブエージェント / フック / MCP / 権限）が **それぞれ何のためにあり、いつ使うか**
- 機能が似ていて迷ったときの **住み分けの判断軸**
- このハンドブックを **どの順で読むと実用的に積み上がるか**

この章は地図です。細かい作り方は各章でやります。まず「どこに何があるか」を頭に入れます。

## 前提

- Claude Code を一度でも触ったことがあること。それだけで十分です。
- 「LLM は次の単語を確率で予測しているだけ」という性質を知っていると、なぜ各拡張点が要るのかが腑に落ちます（旧版 Chapter 1 が `extras/archive/old-chapters/` にあります）。

## 本文

### 1. Claude Code は「頭 ＋ 手足 ＋ 育てしろ」

Claude Code の中身は **LLM（頭）** に、ファイルを読む・コマンドを動かす・外部サービスを触るといった **ツール（手足）** を付けたものです。素のままでも、コードを書いたり調べたりできます。

でも、毎回同じことを説明したり、決まった手順を手で打ち直したりするのは無駄です。そこで Claude Code には、**自分の使い方に合わせて育てるための6つの拡張点**が用意されています。<mark>**この6つを押さえることが「完璧に使いこなす」の地図**</mark>になります。

### 2. 拡張点6つの地図

| # | 拡張点 | 一言で言うと | いつ使う | どこに置く | 本書の章 |
|---|---|---|---|---|---|
| 1 | **CLAUDE.md** | プロジェクトの**記憶**。毎回自動で読まれる前提・ルール | 毎回同じ前提を伝えたい | `CLAUDE.md`（プロジェクト直下 / `~/.claude/`） | 第2章 |
| 2 | **スラッシュコマンド・スキル** | 決まった**手順**を一発で呼ぶ | 繰り返す作業をコマンド化したい | `.claude/commands/*.md` / `.claude/skills/<名前>/SKILL.md` | 第4章 |
| 3 | **サブエージェント** | 別の頭に**役割分担**させる | 重い/独立した作業、別の視点が欲しい | `.claude/agents/*.md` | 第5章 |
| 4 | **フック（hooks）** | 操作の**前後に自動処理**を挟む | 特定操作のたびに自動で何かしたい | `settings.json` の `hooks` | 第6章 |
| 5 | **MCP** | 外部サービスへの**手足を増やす** | Notion・Slack 等を触りたい | `.mcp.json` / `claude mcp add` | 第7章 |
| 6 | **権限（settings.json）** | 自動実行の**安全範囲**を決める | どこまで確認なしで許すか線引き | `settings.json` の `permissions` | 第3章 |

それぞれ、しょうへいさんが**すでに触れているもの**で例えると一気に身近になります。

- **CLAUDE.md** … いつも書いている「推測禁止ルール」「送信系は事前確認」。あれを毎回言わずに済ませる常駐メモ
- **スラッシュコマンド・スキル** … `/zenn-log`（学習ログを Zenn に投稿）、`/save`（Vault に仕分け保存）。「いつもの手順」をコマンド1つに
- **サブエージェント** … **MAGI**（MELCHIOR / BALTHASAR / CASPER の3賢者に別々に見せて多数決）。1人の意見より3人に分担させて事故を減らす
- **フック** … 「ファイルを保存したら自動で整形」「セッション開始時に今の状況を読み込む」のような自動の段取り
- **MCP** … Notion / Slack / Gmail / Google Calendar 連携。Claude の手足を外部サービスまで伸ばす
- **権限** … `settings.json` の `allow` / `deny` / `ask`。「これは確認なしでOK、これは毎回聞いて」の線引き

> [!CAUTION]
> 6つは**競合しません。役割が違う**だけです。「コマンドとスキルどっちで作る？」のように迷うのは正常で、第4〜7章でその判断軸を1つずつ潰します。今は<mark>**「やりたいことに応じて引き出しが6つある」**</mark>とだけ掴めば十分です。

### 3. 2つの軸で整理する

6つは「**いつ効くか**」「**誰が起動するか**」の2軸で見ると、頭の中で並びます。

**軸A：常駐 か 都度か**

- <mark>**常駐（黙っていても効く）**</mark>: CLAUDE.md（毎回読まれる）/ 権限（毎操作で判定）/ フック（条件が合えば自動発火）/ MCP（接続中ずっと使える）
- <mark>**都度（呼んだときだけ）**</mark>: スラッシュコマンド・スキル / サブエージェント

**軸B：自動 か 手動か**

- **手動で起動**: `/zenn-log` のように自分で打つコマンド
- **Claude が自動で起動**: スキルやサブエージェントは、説明文（description）を手がかりに Claude が「これ使えそう」と判断して自分から呼ぶこともある

この「自動で呼ばれる」性質があるからこそ、**説明文（description）を具体的に書くこと**が後の章で繰り返し効いてきます（曖昧だと呼ばれない／余計なときに呼ばれる）。

### 4. 迷ったときの選び方（判断フロー）

やりたいことから逆引きできるようにしておきます。

```
毎回同じ前提・ルールを伝えたい
        └─→ CLAUDE.md（第2章）

決まった手順をワンコマンドで呼びたい
        └─→ スラッシュコマンド／スキル（第4章）

重い・独立した作業を切り出したい／別の視点で検証したい
        └─→ サブエージェント（第5章）  ← MAGI がこれ

「○○したら必ず△△する」を自動で挟みたい
        └─→ フック（第6章）

Notion・Slack など外部サービスを操作したい
        └─→ MCP（第7章）

どこまで確認なしで自動実行を許すか決めたい
        └─→ 権限／settings.json（第3章）
```

> [!CAUTION]
> ここで挙げた**置き場所やファイル名・機能の細部は、Claude Code のバージョンで変わります**。本書は各章末に**公式ドキュメントへのリンク**を必ず置きます。<mark>**「仕組みの理解」は古びませんが、「操作の細部」は古びる**</mark>ので、最新は一次情報で確認してください。

### 5. このハンドブックの進み方

実用的に積み上がる順に並べてあります。**土台 → 道具を作る → 外とつなぐ** の流れです。

| 段階 | 章 | 到達点 |
|---|---|---|
| 導入 | 第0章（今ここ） | 6つの拡張点の地図が頭に入る |
| 初級 | 第1章 土台 | 基本操作・モード（plan mode 等）・コンテキストの考え方 |
| 初級 | 第2章 記憶 | CLAUDE.md とメモリの書き方（3階層） |
| 初級〜中級 | 第3章 権限 | settings.json で自動実行を安全に絞る |
| 中級 | 第4章 コマンド・スキル | 繰り返しをワンコマンドに（/zenn-log・/save が実例） |
| 中級 | 第5章 サブエージェント | 役割分担と多数決（MAGI が実例） |
| 中級〜上級 | 第6章 フック | 操作の前後に自動処理を挟む |
| 上級 | 第7章 MCP | 外部サービスと連携する |
| 総括 | 第8章 日常運用 | 全部を個人開発に組み込む |

第3章（権限）を初級の早めに置いているのは、<mark>**「自動化する前に、暴走しない安全装置を理解しておく」**</mark>のが実用上いちばん効くからです。コマンドやフックで自動実行を増やすほど、権限設計の重要さが効いてきます。

## つまずきポイント

> [!CAUTION]
> 地図を読むときに誤解しがちな点。

- **「どれか1つを選ぶ」ものではない**: 実運用では6つを**組み合わせ**ます。例：MAGI（サブエージェント）を `/magi`（スラッシュコマンド）で呼び、結果を権限（settings.json）で守る、という具合
- **コマンドとスキルは地続き**: ざっくり「決まった手順をまとめたもの」で同じ仲間です。違いと使い分けは第4章で扱うので、今は1つの引き出しと考えてOK
- **「常駐」は便利だが、書きすぎると重い**: CLAUDE.md は毎回読まれるぶん、長いほど毎セッションのコストが増えます（第2章で「150行ルール」を扱います）。常駐させる情報は厳選する
- **細部はバージョンで変わる**: ファイル名・フォルダ位置・フラグは更新されます。本書は核心を教え、細部は公式リンクへ誘導します
